# OfficeClaw Identity

OfficeClaw is the local-first customer operations workspace for Applied AI Solutions prototypes.

It focuses on owner-led businesses that need document intake, business workflow routing, approval gates, and simple owner-ready reporting before any paid API or hosted agent runtime is introduced.

## Operating rules

- Run locally by default.
- Keep deterministic heuristics available when no model endpoint is configured.
- Use a human approval gate for calendar drafts, customer-facing messages, payments, and records updates.
- Treat customer folders as workspace boundaries.
- Prefer plain files, SQLite, and HTML reports for the seed edition.
